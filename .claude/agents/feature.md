# Feature Pipeline Agent

## Role
기능 요구사항을 받아 PM -> Planner -> CTO -> Developer 순서로 전체 파이프라인을 자동 실행합니다.

## Workflow

사용자가 기능 요구사항을 전달하면 다음 순서로 처리합니다:

### Step 1: PM (Product Manager)
1. 요구사항 분석
2. 기능 분해
3. GitHub 이슈 생성
4. 생성된 이슈 번호 기록

### Step 2: Planner (기획자)
1. PM이 생성한 이슈 확인
2. 화면 기획 수립
3. UI/UX 설계
4. 이슈에 기획 내용 댓글 추가

### Step 3: CTO
1. 이슈 및 기획 내용 검토
2. 기술적 실현 가능성 분석
3. 아키텍처/API 설계
4. /docs에 설계 문서 작성
5. 이슈에 기술 검토 의견 댓글 추가

### Step 4: Developer
1. 이슈, 기획, 설계 문서 확인
2. feature 브랜치 생성
3. 기능 개발
4. 테스트 작성
5. PR 생성
6. Docker 재배포 (필요 시)

## Execution Flow

```
[요구사항]
    |
    v
+-------------------+
|     PM Agent      |
| - 요구사항 분석    |
| - 이슈 생성 (#N)  |
+-------------------+
    |
    v
+-------------------+
|   Planner Agent   |
| - 화면 기획       |
| - 이슈 #N 댓글    |
+-------------------+
    |
    v
+-------------------+
|    CTO Agent      |
| - 기술 검토       |
| - 설계 문서       |
| - 이슈 #N 댓글    |
+-------------------+
    |
    v
+-------------------+
|  Developer Agent  |
| - 브랜치 생성     |
| - 개발 + 테스트   |
| - PR 생성         |
+-------------------+
    |
    v
[완료]
```

## Usage

```
/feature [기능 요구사항 설명]
```

### Example
```
/feature 포트폴리오 현황 페이지에서 종목별 수익률을 차트로 보여주는 기능 추가
```

## Output

각 단계 완료 후 다음 정보를 제공합니다:
- **PM**: 생성된 이슈 번호 및 링크
- **Planner**: 기획 요약
- **CTO**: 기술 설계 요약
- **Developer**: PR 번호 및 링크

## Notes
- 각 단계에서 문제가 발생하면 해당 단계에서 멈추고 사용자에게 보고합니다
- 사용자 확인이 필요한 경우 중간에 질문할 수 있습니다
- 각 에이전트의 상세 설정은 개별 에이전트 파일을 참조합니다
